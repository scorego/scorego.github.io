Java集合(三) ArrayList

发表于 2019-05-29 | 分类于 数据结构

1. ArrayList

ArrayList是一个数组列表，相当于动态数组。与数组相比，它的容量能动态增长。它继承于AbstractList抽象类，实现了List、RandomAccess、Cloneable、java.io.Serializable接口。

  • ArrayList继承了AbstractList类，实现了List接口，并提供了添加、删除、修改、遍历等功能。
  • ArrayList实现了RandomAccess接口，即提供了随机访问功能。RandomAccess是java中用来标记支持快速随机访问功能的接口。快速访问功能即通过元素的序号快速获取元素对象。
  • ArrayList实现了Cloneable接口，即覆盖了函数Object.clone()，能被克隆。
  • ArrayList实现了java.io.Serializable接口，意味着ArrayList支持序列化，能通过序列化去传输。

ArrayList不是线程安全的！ 所以建议在单线程中使用ArrayList，多线程中可以选择Vector或者CopyOnWriteArrayList。

ArrayList包含了两个重要的域：Object[] elementData和int size：

  • elementData是Object[]类型的数组，保存了添加到ArrayList的元素。这是一个动态数组，能通过构造函数ArrayList(int initialCapacity)来设置初始容量；如果是无参构造函数，默认容量是10。数组大小会根据ArrayList容量的增长而动态增长，具体增长方式为ensureCapacity()函数。
  • size是列表中元素的实际数量。

1.5 遍历

ArrayList支持4种遍历方式，首先新建一个ArrayList:

  1. 通过迭代器(即Iterator)遍历

  1. 随机访问，通过索引遍历

  2. foreach遍历

  3. Stream遍历

遍历ArrayList时，使用随机访问(即通过索引序号访问)效率最高，而使用foreach、Stream的效率最低。

1.6 toArray()异常

ArrayList提供了2个toArray()函数：

调用ArrayList中的toArray()时可能抛出java.lang.ClassCastException异常，但是调用toArray(T[] a)能正常返回T[]。
toArray()会抛出异常是因为toArray()返回的是Object[]数组，将Object[]转换为其它类型(如如，将Object[]转换为Integer[])则会抛出java.lang.ClassCastException异常，因为Java不支持向下转型。
解决该问题的办法是调用<T> T[] toArray(T[] a) ，而不是Object[] toArray()。
