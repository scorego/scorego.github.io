发表于 2019-05-31 | 更新于 2019-06-11 | 分类于 Java集合

LinkedList

LinkedList是一个继承于AbstractSequentialList的双向链表，也可以当作栈、队列或双端队列进行操作。

  • LinkedList实现了List接口，能对它进行列表操作。
  • LinkedList实现了Deque接口，能将它当做双端队列使用。
  • LinkedList实现了Cloneable接口，即覆盖了函数clone()，能克隆。
  • LinkedList实现了java.io.Serializable接口，意味着它支持序列化，能通过序列化传输。
  • LinkedList是非线程安全的。

1. LinkedList数据结构

继承关系：

LinkedList的本质是双向链表：

  1. LinkedList继承于AbstractSequentialList，并且实现了Dequeue接口。
  2. LinkedList包含两个重要的成员：first和last。
    • first是双向链表的表头，指向第一个结点；last指向最后一个结点。它们的类型是内部类Node<E>。
    • size是双向链表中节点的个数。

AbstractSequentialList类

AbstractSequentialList继承于AbstractList，提供了一个基本的List接口实现，为实现序列访问的数据储存结构的提供了所需要的最小化的接口实现。AbstractSequentialList实现了get(int index)、set(int index, E element)、add(int index, E element) 和remove(int index)这些函数，它们都是提供随机访问的。

LinkedList也实现了List接口，也就是说，它实现了get(int index)、remove(int index)等随机访问方法。LinkedList的随机访问是通过一个计数索引值来实现的。当调用get(int index)时，首先会比较index和”链表长度的一半”，若前者大，则从链表头开始往后查找index次；否则，从链表末尾开始向前查找。LinkedList的顺序访问比较高效，而随机访问效率比较低。

遍历LinkedList时，使用removeFist()或removeLast()效率很高，但它们会删除原始数据。
无论如何，千万不要通过随机访问去遍历LinkedList，从源码来看，使用随机访问遍历LinkedList，时间复杂度是O(n2)。
