HashMap

HashMap是一个散列表，它存储的是键值对(key-value)映射。基本上其与HashTable差不多，除了它不是线程安全的且允许key为null。HashMap不保证映射的顺序，也不保证这些顺序不会改变。

1. 数据结构

继承结构：

HashMap继承于AbstractMap，实现了Map、Cloneable、java.io.Serializable接口。

HashMap的实例有两个参数影响其性能：初始容量initialCapacity和装载因子loadFactor。容量是哈希表中桶的数量，初始容量只是哈希表在创建时的容量。装载因子是哈希表扩容的尺度。当哈希表中的条目数超出了加载因子与当前容量的乘积时，则要对该哈希表进行 rehash操作。
DEFAULT_LOAD_FACTOR是0.75f, 这是在时间和空间成本上寻求一种折衷。装载因子过高虽然减少了空间开销，但同时也增加了查询成本。在设置初始容量时应该考虑到映射中所需的条目数及其装载因子，以便最大限度地减少rehash操作次数。如果初始容量大于最大条目数除以加载因子，则不会发生rehash操作。
