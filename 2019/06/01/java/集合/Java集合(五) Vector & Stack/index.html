Java集合(五) Vector & Stack

发表于 2019-06-01 | 更新于 2019-06-02 | 分类于 Java集合

1. Vector

Vector是矢量队列，继承于AbstractList，实现了List、RandomAccess、Cloneable、java.io.Serializable接口。

  • Vector继承了AbstractList，实现了List；所以，它是一个队列，支持相关的添加、删除、修改、遍历等功能。
  • Vector实现了RandmoAccess接口，即提供了随机访问功能。
  • Vector实现了Cloneable接口，即实现clone()函数。它能被克隆。

Vector中的操作是线程安全的。

1.1 数据结构

继承关系：

Vector的数据结构和ArrayList差不多，它包含了3个成员变量：Object[] elementData,int elementCount，int capacityIncrement。

  • elementData是Object[]类型的数组，它保存了添加到Vector中的元素。elementData是个动态数组，如果初始化Vector时，没指定动态数组的大小，则使用默认大小10。随着Vector中元素的增加，Vector的容量也会动态增长，capacityIncrement是与容量增长相关的增长系数，具体的增长方式查看源码的ensureCapacityHelper()函数。
  • elementCount是动态数组的实际大小。
  • capacityIncrement是动态数组的增长系数。如果在创建Vector时指定了capacityIncrement的大小；则每次当Vector中动态数组容量增加时，增加的大小都是capacityIncrement(只在capacityIncrement大于0时生效；capacityIncrement小于等于0时，新数组长度增加为原来数组长度的2倍)。

2. Stack

Stack是栈，即先进后出(FILO, First In Last Out)。
Stack是继承于Vector的，这就意味着Stack也是通过数组实现的，而非链表。当然也可以将LinkedList当作栈来使用。

2.4 源码分析

Stack继承于Vector，意味着Vector拥有的属性和功能，Stack都拥有。
Stack是通过数组去实现的,执行push()时(即入栈)，是通过将元素追加的数组的末尾中。执行peek时(即，取栈顶元素，而不删除)，是返回数组末尾的元素。执行pop时(即出栈)，是取出数组末尾的元素，然后将该元素从数组中删除。
