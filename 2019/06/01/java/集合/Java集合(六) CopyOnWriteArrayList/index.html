Java集合(六) CopyOnWriteArrayList

发表于 2019-06-01 | 更新于 2019-06-03 | 分类于 Java集合

CopyOnWrite，简写COW,即写时复制，基本思路是对于共享的同一个内容，当需要修改这个内容的时候，会把内容Copy一份然后再改，这是一种延时懒惰策略。从JDK1.5开始JUC包里提供了两个使用CopyOnWrite机制实现的并发容器,它们是CopyOnWriteArrayList和CopyOnWriteArraySet。

CopyOnWriteArrayList

CopyOnWriteArrayList是ArrayList线程安全的变种，其所有的可变操作(add()、set()等)都是通过复制底层数组来实现的。一般来说这种实现开销很大，但是当遍历操作远大于改变操作时，这种实现比替代方法更有效。而且当不能或不希望同步遍历，但又需要排除并发线程之间的干扰时，这种实现非常有用。“快照”样式迭代器(iterator)使用的是迭代器被创建那一刻的数组的引用。该数组在迭代器的生命周期内从未更改，因此不可能发生并发干扰，并且迭代器不会抛出ConcurrentModificationException。一旦迭代器创建，它就无法反映迭代器创建后对列表的更改操作；同时迭代器也不支持更改操作，这会抛出UnsupportedOperationException。
一致性:与其他并发集合一样，在将对象放入CopyOnWriteArrayList之前的线程中的操作发生在从另一个线程的CopyOnWriteArrayList中访问或删除该元素之后的操作之前(happen-before)。

CopyOnWriteArrayList需要特别关注的是迭代器的创建方法。CopyOnWriteArrayList的内部类COWIterator继承于ListIterator，内部有一个final的域Object[] snapshot。当COWIterator的构造方法被调用时，snapshot指向此刻CopyOnWriteArrayList的Object[] array。而且原列表的更新操作只会用一个拷贝后的新数组替换原数组，所以迭代器初始化之后对列表的更新操作并不会影响此迭代器的遍历。
同时也要注意到此迭代器的更新操作一律抛出UnsupportedOperationException。
