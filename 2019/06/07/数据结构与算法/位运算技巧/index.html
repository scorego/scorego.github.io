位运算技巧

发表于 2019-06-07 | 更新于 2019-06-08 | 分类于 数据结构与算法

由于位运算直接对内存数据进行操作，因此非常高效。另外一些位运算也十分赏心悦目，因此总结了一些常用的位运算技巧笔记。

位运算基础

  • 与运算 &
    与运算符&是双目运算符。只有对应的两个二进制位都是1时，结果位才是1,；否则是0.

  • 或运算 |
    或运算符是双目运算符。只要对应的两个二进制位有一个为1时，结果位就是1；否则是0。

  • 非运算 ~
    非运算符~是单目运算符。功能是按位取反。

  • 异或 ^
    异或运算符^是双目运算符。对应的两个二进制位相同为0，相异为1。

  • 左移<<、右移>>(一些语言包括无符号右移>>>)
    左移就是向左移动k位，丢弃掉最高的k位，并在右端补0。也就是当前值乘以2的k次方。
    右移分逻辑右移和算术右移。逻辑右移补0，算术右移补符号位。

1. 消去x的二进制中最后一个1

如，10的二进制是1010， 1010 & 1001 = 1000.

  • a. 可以用来检测一个数是不是2的幂
    如果一个数是2的幂，那么x > 0且x的二进制数中只有一个1，所以x & (x - 1)应该返回0。

  • b. 用来计算一个整数的二进制中1的个数
    因为1可以不断的通过x & (x - 1)消去，所以当最后的值变成0的时候，也就求出了二进制中1的个数。

  • c. 如果将整数A转换成整数B，需要改变多少个比特位
    也就是计算A ^ B中的二进制数中共有多少个1.

3. 利用二进制来枚举子集

子集的概念： 如果集合A的任意一个元素都是集合B的元素，那么集合A称为集合B的子集。
在写程序的时候，有时候我们可能需要暴力枚举出所有的情况，这时可以考虑通过二进制来枚举子集来尝试解决问题。
示例：
假设我们现在有5个小球，上面分别标号了0,1,2,3,4代表这些小球的权值,求出这些小球的权值可以组成的所有情况。
我们可以用5个比特位来表示这种情况，如果小球全部选择的话那么二进制表示就是11111，二进制的11111转化为十进制数字就是31，这个数字正好就是2 ^ 5 - 1，那么我们可以用从0 ~ (2 ^ 5−1)这些数表示完所有的选取状态(因为这个范围内的二进制数情况正好包括了这些选取状况)。

4. 一些经典应用

  • i + (~i) = -1
    i取反再与i相加，相当于把所有二进制位设为1，其十进制结果为-1。

  • 计算n + 1 和 n - 1
    -~n == n + 1，~n为其取反，负号-再对其取反并加1。
    ~-n == n - 1，思路就是找到最低位的第一个1，对其取反并把该位后的所有位也取反，即01001000变为01000111。

  • 取相反数
    思路就是取反并加1，也即 ~n + 1或者(n ^ -1) + 1。

  • if(x == a) x = b; if(x == b) x = a
    利用^运算符的性质，即得x = a ^ b ^ x。

  • n倍数补全
    当n为2的幂时，(x + n - 1) & ~(n - 1)会找到第一个大于x的数，且它正好是n的整数倍。

其中n >> 31取得n的正负号。
若n为正数，n >> 31的所有位等于0，其值等于0。表达式转化为n ^ 0 - 0，等于n；
若n为负数，n >> 31的所有位等于1，其值等于-1。表达式转化为(n ^ -1) + 1，这很好理解，负数的相反数就是对其补码取反再加1，(n ^ -1) + 1就是在做这样的事。
