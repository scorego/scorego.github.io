Java并发07 ReentrantLock

发表于 2019-06-09 | 更新于 2019-06-10 | 分类于 Java并发

ReentrantLock是一个可重入的互斥锁，具有和synchronized相同的互斥性和内存可见性，还实现了一些额外功能。synchronized大多数情况都能很好的工作，但功能上存在一些局限，如无法中断一个正在等待获取锁的线程、无法在请求获取一个锁时无线等待下去、无法实现非阻塞结构的加锁规则(因为其必须在获取锁的代码块中释放锁)。 、
可重入锁可以理解为锁的一个标识。该标识具备计数器功能。标识的初始值为0，表示当前锁没有被任何线程持有。每次线程获得一个可重入锁的时候，该锁的计数器就被加1。每次一个线程释放该所的时候，该锁的计数器就减1。前提是：当前线程已经获得了该锁，而且是在线程的内部出现再次获取锁的场景。
ReentrantLock可以通过构造方法参数设置为”公平锁”和”非公平锁”。公平锁在加锁时倾向于等待时间最长的线程，非公平锁不保证获得锁的顺序。在多线程时公平锁的吞吐量比非公平锁低(通常低很多)，但在获得锁的时间上有较小的方差，并且保证不出现饥饿现象。但注意，公平锁并不保证线程调度的公平性。 还要注意，非定时的tryLock()方法不支持公平设置。如果锁可用，即使其他线程正在等待，它也会成功。
