数据结构03 二叉树

发表于 2019-06-11 | 更新于 2019-06-14 | 分类于 数据结构与算法

树是一种半线性结构，是n个结点的有限集合。树适合于表示具有层次结构的数据。树中的某个结点最多只和上一层的一个节点有直接关系(根节点除外)，和下一层的零个或多个结点有直接关系。

1. 基本概念

树中一个结点的子结点个数称为该结点的度。树中结点的最大度数称为树的度。显然，度为0的节点是叶子结点。
结点的深度是从根节点开始自顶向下累加，结点的高度是从叶节点开始自底向上逐层累加。树的高度是树中结点的最大层数。

树中的结点树等于所有结点的度数加1。
度为m的树中的第i层上最多有mi - 1个结点。
高度为h的m叉树最多有(mh - 1)/(m - 1)个结点。

2. 二叉树

二叉树(Binary tree)中，每个节点的度都不能超过2。二叉树是有序树，即使树中结点只有一棵子树，也要区分它是左子树还是右子树。
所有的结点都只有左子树的二叉树叫左斜树；所有结点都是只有右子树的二叉树叫右斜树。
如果一棵二叉树的左子树上所有结点的值均小于根节点的值，且右子树的所有节点的值均大于根节点的值，且左子树和右子树也符合这个条件，就称这棵树是二叉排序树(二叉查找树) 。
如果树上任一结点的左子树和右子树的深度之差不超过1，就称这棵二叉树是平衡二叉树。

2.1 存储结构

二叉树可以使用顺序存储结构和链式存储结构。
顺序存储结构用一组地址连续的存储单元一次自上而下、自左至右存储完全二叉树上的结点元素。对于满二叉树和完全二叉树，树中结点序号可以唯一反应结点之间逻辑关系，且能最大程度节省存储空间；但对于一般二叉树，只能添加一些并不存在的空节点。
由于顺序存储的空间利用率较低，所以一般使用链式存储。二叉树的每个结点用指针来指向自己的子结点。容易验证，一个n结点的二叉链表中含有n + 1个空指针，可以利用这些空指针来组成另一种链表结构：线索链表。

2.2 性质

性质1：非空二叉树上的叶子节点数等于度为2的结点树加1。即n0 = n2 + 1。

证明：设度为0/1/2的结点个数分别是n0、n1、n2，则结点总数n = n0 + n1 + n0。又有除根节点外，其他结点都有父结点，故n = n1 + 2n2 + 1。则n0 + n1 + n0 = n1 + 2n2 + 1，推出n0 = n2 + 1。

性质2：非空二叉树第k层上至多有2k - 1个结点。(k >= 1)

性质3：高度为h的二叉树至多有2h - 1个结点。





2.3 二叉树的遍历

遍历是二叉树各种操作的基础，可以在遍历的过程中对结点进行各种操作。

A. 前/中/后序遍历

通常，实现二叉树的前序(preorder, NLR)、中序(inorder, LNR)、后序(postorder, LRN)遍历有两个常用的方法：递归和栈。这两种方法的空间复杂度和时间复杂度都是一样的：每个结点都访问且只访问一次，所以时间复杂度是O(n)；递归栈的深度是树的深度，所以最坏情况是一个斜树，空间复杂度是O(n)。
