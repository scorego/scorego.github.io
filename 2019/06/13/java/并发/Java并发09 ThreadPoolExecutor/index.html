3. 一些域

ctl是线程池的状态(control state)，是一个AtomicInteger类。它实际上是将两个域整合为一个原子整形，这两个域是:

  • workerCount，表示有效的线程数量
  • runState，指示状态

一个整形是32位，为了将两个参数整合到一个，用高3位来表示runState，低29位表示workerCount。所以实际的workerCount最大值是229 - 1，而不是231 - 1(int的第一位是符号位)。计算ctl的方法为：

COUNT_BITS = Integer.SIZE - 3，也就是说COUNT_BITS是29。CAPACITY是线程池的最大数量，为CAPACITY = (1 << COUNT_BITS) - 1，也就是说CAPACITY的高3位是0，低29位全是1。
线程池的运行状态有:

  • RUNNING: Accept new tasks and process queued tasks
  • SHUTDOWN: Don’t accept new tasks, but process queued tasks
  • STOP: Don’t accept new tasks, don’t process queued tasks, and interrupt in-progress tasks
  • TIDYING: All tasks have terminated, workerCount is zero, the thread transitioning to state TIDYING will run the terminated() hook method
  • TERMINATED: terminated() has completed

由于CAPACITY的高3位是0，低29位全是1，所以方法int runStateOf(int c) { return c & ~CAPACITY; }实际上是取ctl的高3位；int workerCountOf(int c) { return c & CAPACITY; }是取低29位；int ctlOf(int rs, int wc) { return rs | wc; }就是将rs和rc拼起来(因为rs的低29位全是0)。
