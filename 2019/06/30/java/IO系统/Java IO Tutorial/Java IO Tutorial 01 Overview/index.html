Java IO Tutorial 01 Overview

发表于 2019-06-30 | 更新于 2019-07-13 | 分类于 Java IO

Java IO Tutorial
Java Networking API

Java IO API位于Java IO包java.io中，包中有大量的类。java.io包实际上并不涉及所有类型的输入和输出，比如不包含对GUI或Web页面的输入和输出(这些类型的输入在其他地方有所涉及，例如Swing项目中的JFC类，或Java Enterprise Edition中的Servlet和HTTP包)。Java IO包主要关注文件、网络流、内部存储器缓冲区等的输入和输出。但是，Java IO包不包含用于打开网络通信所必需的网络套接字的类，为此需要使用Java Networking API建立连接。一旦建立了一个socket连接(网络连接)，就可以通过Java IO InputStream和OutputStream类来读取和写入数据。

1. Overview

A. 信源和信宿

Java的IO包主要关注从信源读取原始数据和将原始数据写入信宿。最典型的信源和信宿是：

  • 文件
  • 管道
  • 网络连接
  • 内存缓冲区(如数组)
  • System.in, System.out, System.error

Source >>> Program >>> Destination

stream是Java IO的核心概念。stream概念上是无穷无尽的数据流，可以从流中读取数据或写数据到流中。流连接到信源或信宿。Java IO中的流可以是基于字节(byte)的(读取和写入字节)，也可以是基于字符(character)的(读取和写入字符)。
从某个信源读取数据的程序需要一个InputStream或一个Reader，将数据写入某个信宿的程序需要OutputStream或Writer。InputStream或Reader链接到信源，OutputStream或Writer链接到信宿。

Source >>> InputStream/Reader >>> Program
Program >>> OutputStream/Writer >>> Destination

B. Java IO目的和功能

Java IO有很多InputStream/OutputStream/Reader/Writer的子类，不同的子类解决不同的问题，包括：

  • Files
  • Network
  • 访问内置存储
  • 线程间通信(管道)
  • Buffering
  • Filtering
  • Parsing
  • Reading and Writing Text (Readers / Writers)
  • Reading and Writing Primitive Data (long, int etc.)
  • Reading and Writing Objects

2. Files / 文件

A. 读取文件

从文件中读取内容时可以使用FileInputStream/FileReader，它们每次读取一个字节或字符，或读入一个字节和字符数组。不必每次读取文件的全部内容，但必须按顺序读取。如果需要不按顺序读取，可以使用RandomAccessFile。

B. 编写文件

向文件写入内容可以使用FileOutputStream/FileWriter，必须按顺序写入。如果需要不按顺序写入，比如追加内容到文件末尾，可以使用RandomAccessFile。

C. 随机访问文件

可以通过RandomAccessFile类随机访问文件。随机访问并不意味着从真正随机的地方读取或写入，而是意味着可以以任何方式跳过文件并同时读取或写入文件。没有强制执行特定的访问顺序。这样就可以覆写现有文件的一部分、追加内容到末尾、从中删除、从需要的地方读取。

D. 文件和目录信息访问

有时可能需要访问有关文件的属性而非其内容，如有时需要知道文件的大小或其他属性。对于目录也是如此，如可能希望获取给定目录中所有文件的列表。文件和目录信息都可以通过File类获得。

3. Pipes / 管道

Java IO的管道提供了在同一JVM中运行的两个线程通信的能力。所以管道既是信源，又是信宿。
不能使用管道向不同JVM的线程通信。Java中的管道的概念和UNIX/Linux中管道的概念是不同的。UNIX/Linux中，在不同地址空间中运行的两个进程都可以通过管道进行通信。在Java中，通信方必须在同一进程中运行，并且应该是不同的线程。
Java中通过PipedOutputStream/PipedInputStream类来创建管道。一个PipedInputStream应该连接到一个PipedOutputStream。一个线程写入PipedOutputStream的数据可以被另一个线程从PipedOutputStream读取。下面是一个示例：

除了管道之外，还有许多其他方法可以在同一个JVM中进行通信。事实上，线程更经常交换完整的对象而不是原始的字节数据。但是如果需要在线程之间交换原始字节数据，Java IO的管道是可选的。

6. System.in, System.out, and System.error

System.in、System.out和System.err也是常见的信源和信宿。这三个流在JVM启动时初始化。
System.in是一个InputStream，通常连接到控制台程序的键盘输入。由于数据通常通过命令行参数或配置文件传递到命令行Java程序，因此System.in使用并不频繁。在具有GUI的应用程序中，应用程序的输入是通过GUI给出的。这是Java IO的独立输入机制。
System.out是一个PrintStream，通常向控制台写数据。这通常用于仅限控制台的程序，如命令行工具。有时也用于从程序中打印调试语句(尽管它可能不是从程序中获取调试信息的最佳方法)。
System.err是一个PrintStream。除了它通常用于输出错误文本外，它和System.out相同。某些程序(如Eclipse)以红色文本显示System.err输出，以表明其是错误文本。

7. Stream

Java IO流是可以读取或写入的数据流。流通常连接到数据源或数据目标，如文件，网络连接等。流的读取或写入没有数组那样索引的概念。通常也不能在流中前后移动，流只是一个连续的数据流。
一些流实现如PushbackInputStream可以将数据推回到流中，以便稍后再次重新读取。但是只能推回有限数量的数据，并且不能随意遍历数据。数据只能按顺序访问。
Java IO流通常要么基于字节，要么基于字符。基于字节的流通常被称为stream，如InputStream或OutputStream。这些流每次读取和写入的是原始字节，但是DataInputStream/DataOutputStream还可以读取和写入int/long/float/double值。基于字符的流通常称为Reader或Writer，可以读/写字符(如Latin1或UNICODE字符)。

10. Concurrent IO

有时可能需要并发处理输入和输出，也就是说可能需要不止一个线程来处理输入输出。比如如果需要处理很多文件，并行处理会带来很大的性能提升。如果需要并发IO，需要注意以下几个常见问题：

  • 同一时刻不应该有多个线程从一个InputStream或Reader读取数据，也不应该有多个线程同时写入OutputStream或Writer。如果这么做，将无法保证每个线程读取多少输入，或者线程将数据写入输出流时的顺序。
  • 如果按顺序使用stream/reader/writer，则可以有多个线程。例如，您可能有一个线程从输入中确定它是什么类型的请求，然后将其交给相应的线程进行进一步处理。这是可行的，因为对stream/reader/writer的访问是顺序的。注意，应该正确地同步在线程之间移交流的代码。

注意：在Java NIO中，一个线程可以读取/写入多个channel。例如，如果打开了大量网络连接，但每个连接上只有很少的数据(如在聊天服务器中)，可以使用单个线程监视所有通道(连接)。
