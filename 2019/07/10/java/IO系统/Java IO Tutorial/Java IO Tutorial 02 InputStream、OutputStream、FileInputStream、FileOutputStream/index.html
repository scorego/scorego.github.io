Java IO Tutorial 02 InputStream、OutputStream、FileInputStream、FileOutputStream

发表于 2019-07-10 | 更新于 2019-07-11 | 分类于 Java IO

1. InputStream

InputStream类表示有序的字节流。换句话说，可以把从InputStream中读取的数据作为有序的字节序列。这在从文件读取数据或通过网络接收数据时非常有用。InputStream类是Java IO API中所有输入流的基类。InputStream子类包括FileInputStream、BufferedInputStream和PushbackInputStream等。

read()方法返回一个int，可以将其强制转为char。子类有替代read()的方法。例如，DataInputStream的readBoolean()/readDouble()可以读取基本类型如int/long/float/double/boolean等。如果read()方法返回-1，则已到达流的末尾，意味着没有更多数据。也就是说，-1是int值，而不是字节或short。

InputStream还包含两个方法int read(byte[])/int read(byte[], int offset, int length)，可以将信源的数据读入到一个字节数组内，这比一次只读一个字节快得多，所以尽可能用这两个方法。
int read(byte[])一次读入尽可能多的数据到数组中，并返回一个int值，表示实际读取的字节数。如果从数组中读取的字节数少于数组的空间，则数组的其余部分内容保持不变。请记住检查返回的int以查看实际读入数组的字节数。
int read(byte[], int offset, int length)方法也将字节读入byte数组，但从offset字节开始进入数组，并做多读取length字节。同样，该方法返回一个int告知实际读入数组的字节数，因此请记住在处理读取字节之前检查此值。

C. mark()、reset()

InputStream有两个方法mark()、reset()，子类可以支持也可以不支持这两个方法。如果子类支持这两个方法，那么就要重写markSupported()方法，令其返回true。markSupported()返回false表示不支持这两个方法。
mark()在InputStream内设置标记，标记到目前为止流中已读取数据的点。然后代码继续从InputStreamcan中读取数据。代码调用reset()可以回到流中标记的点读取数据，此时该InputStream 则回到标志，并重新开始从该点(读取)数据。这会导致一些数据返回多次。通常在实现解析器时使用这些方法。有时解析器可能需要提前读取InputStream，如果解析器找不到预期的内容，则可能需要回退并尝试将读取的数据与其他内容进行匹配。

2. OutputStream

OutputStream类是Java IO API中所有的输出流的基类。子类包括BufferedOutputStream和FileOutputStream等。一个OutputStream通常总是连接到某些信宿，如文件、网络连接、管道等。信宿是写入OutputStream的所有数据最终的目的地。

C. flush()

OutputStream的flush()方法刷新写入OutputStream的所有数据到信宿。例如，如果OutputStream是FileOutputStream，则写入FileOutputStream的字节可能尚未完全写入磁盘。即使Java代码已将数据写入，FileOutputStream也可能将数据缓存在内存。通过调用flush()可以确保将任何缓冲的数据刷新(写入)到磁盘()或网络等其他目的地)。

3. FileInputStream

FileInputStream类使得可以将文件的内容作为字节流读取。该类是InputStream的子类，意味着可以将FileInputStream用作InputStream。FileInputStream用来读取原始字节，比如图片数据。当读取字符流时，考虑使用FileReader。
