Java NIO Tutorial

Java NIO是Java 1.4新加入的替代IO的API。IO是面向流的，NIO是面向缓冲区的。标准IO API使用的是字节流和字符流。NIO使用通道(Channel)和缓冲区(Buffer)。数据总是从通道读入缓冲区，或从缓冲区写入通道。
NIO也是非阻塞IO(Non-Blocking IO)。线程可以要求通道将数据读入缓冲区。在通道把数据读入缓冲区时，线程可以执行其他操作。一旦数据被读入缓冲区，线程就可以继续处理它。将数据写入通道也是如此。
Java NIO包含选择器(Selector)的概念。选择器是一个可以监视多个事件通道的对象（例如：连接打开，数据到达等）。因此，单个线程可以监视多个通道的数据。

1. Overview

NIO由以下核心组件组成：

  • Channel 通道
  • Buffer 缓冲区
  • Selector 选择器

NIO也有其他的类和组件，但这三个是核心。其余组件比如Pipe/FileLock仅仅是与三个核心组件一起使用的实用程序类。

A. Channel 和 Buffer

通常，NIO的所有IO都是以一个Channel开始的。Channel有点像流，数据可以从Channel读到一个Buffer，也可以从Buffer写入Channel。以下是Channel的主要实现：

  • FileChannel
  • DatagramChannel
  • SocketChannel
  • ServerSocketChannel

这些通道涵盖UDP/TCP Network IO和 File IO。以下是Buffer的核心实现：

  • ByteBuffer
  • CharBuffer
  • DoubleBuffer
  • FloatBuffer
  • IntBuffer
  • LongBuffer
  • ShortBuffer

这些Buffer涵盖了可以通过IO发送的基本数据类型。NIO也有MappedByteBuffer与内存映射文件一起使用。

B. Selector

Selector允许单线程处理多个Channel。如果应用程序打开了许多连接(通道)，但每个连接只有较低的流量，这很方便，如聊天服务器。

2. NIO Channel

NIO通道类似于流，但有一些差异：

  • 可以读取和写入频道。流通常是单向的（读或写）。
  • 可以异步读取和写入通道。
  • 通道始终读取缓冲区或从缓冲区写入。

以下是Channel的主要实现：

  • FileChannel 从/到文件中读取数据
  • DatagramChannel 通过UDP网络读/写数据
  • SocketChannel 通过TCP网络读/写数据
  • ServerSocketChannel 监听TCP连接，对每个连接建立一个ServerSocketChannel

3. NIO Buffer

在与NIO通道交互时使用NIO缓冲区。数据从通道读入缓冲区，或从缓冲区写入通道。缓冲区本质上是一个可以写入数据的内存块，然后可以再次读取。此内存块包含在NIO Buffer对象中，该对象提供了一组方法，可以更轻松地使用内存块。

使用Buffer来读取和写入数据通常遵循以下四个步骤：

  1. 将数据写入缓冲区
  2. 调用buffer.flip()
  3. 从缓冲区读取数据
  4. 调用buffer.clear()或buffer.compact()

将数据写入缓冲区时，缓冲区会跟踪写入的数据量。一旦需要读取数据，就需要使用flip()方法调用将缓冲区从写入模式切换到读模式。在读模式下，缓冲区允许读取写入缓冲区的所有数据。一旦读完所有数据，就需要清除缓冲区，以便再次写入。可以通过两种方式完成此操作：通过clear()或compact()。clear()方法清除整个缓冲区。compact()方法仅清除已读的数据，任何未读数据都会移动到缓冲区的开头，新数据将从未读数据之后写入缓冲区。

6. Selector

NIO Selector是一个可以检查一个或多个Java NIO Channel实例的组件，并确定哪些Channel可以用于读取或写入。这样，单个线程可以管理多个通道，从而管理多个网络连接。
使用Selector的好处是一个只需一个线程即可管理多个连接，这样减少了线程创建与切换的开销。

B. 通过选择器选择通道

如果Selector注册了多个通道，那么可以用select()方法，该方法返回监听事件就绪的通道。select()方法有：int select()、int select(long timeout)、int selectNow()。select()阻塞，直到至少有一个通道为注册的事件就绪。select(long timeout)除了阻塞最多timeout毫秒之外，与select()作用相同。selectNow()完全不会阻塞，它会立即返回任何已准备好的通道。
