Java JDBC Tutorial

Java JDBC API (Java Database Connectivity)使Java应用程序可以连接到关系数据库，比如MySQL/PostgreSQL/MS SQL Server/Oracle/H2 Database等。Java JDBC API是Java SE SDK的一部分。



JDBC是数据库独立的。连接不同的数据库使用的是相同结构的代码，所以如果需要更换数据库是十分方便的。
JDBC并不是SQL独立的。不同数据库的SQL是有略微区别的，所以SQL是和数据库契合的。
JDBC并不是用来连接非关系数据库的。JDBC是通过标准SQL来和数据库交互的。所以JDBC并不能和非关系数据库交互，如MongoDB/Cassandra/Dynamo等。

JDBC的一些核心概念：

  • JDBC Driver
  • Connection
  • Statement
  • PreparedStatement
  • ResultSet
  • CallableStatement
  • Batch Updates
  • Transactions
  • DatabaseMetaData

B. 核心组件

1. JDBC Drivers

JDBC驱动是一些Java类的集合，可以用来连接到某个数据库。例如，MySQL拥有自己的JDBC驱动程序。JDBC驱动程序实现了JDBC接口。当代码使用给定的JDBC驱动程序时，它实际上只使用标准的JDBC接口。使用的具体JDBC驱动程序隐藏在JDBC接口后面。因此，可以在不影响代码的情况下插入新的JDBC驱动程序。当然，不同JDBC驱动程序在它们支持的功能方面可能略有不同。

2. Connections

加载并初始化JDBC驱动程序后，就可以连接到数据库。连接数据库是通过JDBC API和加载的驱动程序获取的数据库Connection。与数据库的所有通信都通过Connection进行。应用程序一次可以打开多个数据库连接。

3. Statements

Statement用来对数据库执行查询和更新。可以使用几种不同类型的语句，每个语句对应一个查询或更新。

4. ResultSets

对数据库执行查询后，会得到一个ResultSet。然后，可以遍历此ResultSet以读取查询结果。

C. 常见的JDBC场景

1. Query the database

非常常见的场景，也即从数据库中读数据。

2. Query the database meta data

另一个常见用例是查询数据库元数据。数据库元数据包含有关数据库本身的信息。例如，有关定义的表的信息、每个表中的列、数据类型等。

3. Update the database

另一个非常常见的JDBC用例是更新数据库。更新数据库意味着将数据写入其中。换句话说，添加新记录或修改（更新）现有记录。

4. Perform transactions

事务将多个更新和查询组合到单个操作中。所有操作要么都已执行，要么都不执行。
