3. Statement

java.sql.Statement接口用来对一个数据库执行SQL语句。Statement是从JDBC Connection获得的。一旦获取Statement，就可以用来对数据库执行检索或更新操作。

4. ResultSet

java.sql.ResultSet是查询返回的结果。ResultSet包含记录。每条记录都包含一组列。每条记录包含相同数量的列，但并非所有列都可能具有值。列可以有null值。
创建ResultSet的时候有三种属性可以设置，分别是：

  • Type
  • Concurrency
  • Holdability

A. Type

ResultSet的类型决定了ResultSet的一些属性。并非所有的数据库和JDBC驱动都支持所有的ResultSet类型，需要查看文档。可以通过DatabaseMetaData.supportsResultSetType(int type)来查看是否支持某一具体类型。共有3种类型：

  • ResultSet.TYPE_FORWARD_ONLY
  • ResultSet.TYPE_SCROLL_INSENSITIVE
  • ResultSet.TYPE_SCROLL_SENSITIVE

默认为ResultSet.TYPE_FORWARD_ONLY，意味着ResultSet只能向前导航，也就是无法回退移动。TYPE_SCROLL_INSENSITIVE表示ResultSet可以向前或向后移动，也可以跳转到相对位置或绝对位置。但如果ResultSet数据库中的记录被另一个线程或进程更改，它将不会反映在已打开的此类型的ResulsSet中。TYPE_SCROLL_SENSITIVE表示ResultSet可以向前或向后移动，也可以跳转到相对位置或绝对位置。但如果ResultSet数据库中的记录被另一个线程或进程更改，它将反映在已打开的此类型的ResulsSet中。

B. Concurrency

并发表示ResultSet是否可写，级别有两种：

  • ResultSet.CONCUR_READ_ONLY 只读
  • ResultSet.CONCUR_UPDATABLE 读写

C. Holdability

holdability表示当connection的commit()方法被调用时，ResultSet是否关闭。有两种：

  • ResultSet.CLOSE_CURSORS_OVER_COMMIT 关闭
  • ResultSet.HOLD_CURSORS_OVER_COMMIT 保持

5. PreparedStatement

PreparedStatement是Statement的子类，具有一些附加功能。PreparedStatement可以：

  • Easy to insert parameters into the SQL statement.
  • Easy to reuse the PreparedStatement with new parameter values.
  • May increase performance of executed statements.
  • Enables easier batch updates.

B. 性能

数据库需要花费一定时间来分析SQL以确定最优的处理方式。如果为每个查询或更新都提交新的完整SQL语句，则数据库必须重新解析SQL。通过重用现有的PreparedStatement可以为后续查询重用SQL解析和查询计划。这可以加速查询执行。
有两个潜在的PreparedStatement重用(reuse)级别：

  • JDBC驱动重用PreparedStatement
  • 数据库重用PreparedStatement

JDBC驱动程序可以在内部缓存PreparedStatement对象，从而重用这些对象。这可以节省一点PreparedStatement的创建时间。其次，对于使用相同数据库的集群，不同实例的Java应用程序可能会重用缓存的解析和查询计划。



6. Batch Updates

JDBC 批量更新是一批处理组合在一起，并在一次发送全部更新，而不是一个一个发送更新。一次性向数据库发送一批更新比一次一个地发送更快。发送一批更新(仅往返1次)涉及的网络流量较少，数据库可能能够并行执行某些更新。与逐个执行更新相比，速度提升可能非常大。可以批量SQL插入，更新和删除。批量检索语句没有意义。可以使用Statement/PreparedStatement来执行批量更新。

注意，每个添加到Statement或PreparedStatement的更新都由数据库单独执行，这一点很重要。这意味着，其中一些可能会在第一个失败之前执行成功。已成功的所有语句现在都应用于数据库，但其余更新可能不适用。这可能导致数据库中的数据不一致。为避免这种情况，可以在JDBC事务中执行批量更新。

7. Transactions
