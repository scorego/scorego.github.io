5. CompletableFuture

Java 8新增了CompletableFuture类，提供了非常强大的Future扩展功能，可以帮助简化异步编程的复杂性，并提供了函数式编程的能力。
继承结构为：

CompletionStage接口代表异步计算中的一个阶段或步骤，该接口定义了多种方法，一般都返回新的CompleteStage，表示执行完一些逻辑后生成新的CompleteStage，构成链式的阶段型的操作。
CompleteFuture代表着一个Future完成后该干点什么，大致有：

  • Future完成后执行动作，活求取下一个Future的值。then…
  • 多个Future的协调。同时完成该怎么做，其中一个完成该怎么做。allOf,anyOf

CompletableFuture有很多方法，并伴有异步和非异步两个版本。Async结尾的方法都是可以异步执行的，如果指定了线程池，会在指定的线程池中执行，如果没有指定，默认会在ForkJoinPool.commonPool()中执行。

有四个静态方法用来为一段异步执行的代码创建CompletableFuture对象，方法的参数类型都是函数式接口，可以使用lambda表达式实现异步任务。runAsync()以Runnabel函数式接口类型为参数，所以CompletableFuture的计算结果为空。supplyAsync()以Supplier<U>函数式接口类型为参数，CompletableFuture的计算结果类型为U。
