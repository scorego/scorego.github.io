计算机网络02 应用层

发表于 2019-09-11 | 更新于 2019-09-13 | 分类于 计算机网络

网络应用是计算机网络存在的理由。研发网络应用程序的核心是写出能够运行在不同的端系统和通过网络批次通信的程序。现代网络应用程序有两种主流体系：客户-服务器体系结构或对等体系结构。
客户-服务端体系结构(client-server architecture)中，有一个总是打开的主机称为服务器，它服务来自客户的请求。服务器具有固定的、周知的地址(IP地址)，客户能够通过地址来向服务器发送分组与其联系。此种体系结构中，客户端主动地发起通信请求，服务器被动地等待通信的建立，客户之间不直接通信。
对等体系结构(P2P architecture)对位于数据中心的专用服务器具有最小的依赖。应用程序在间断连接的主机对之间使用直接通信，这些主机被称为对等方。许多流行的、流量密集型应用都是P2P体系结构，如文件共享(BitTorrent)、对等方协助下载加速器(迅雷)、因特网电话(Skype)等。P2P体系结构的自扩展性比较好，尽管每个对等方都由于请求产生工作量，但每个对等方通过向其他对等方提供服务也为系统增加服务能力。
同一个端系统上的进程通过操作系统的进程间通信机制相互通信，而不同端系统上的进程通过跨越计算机网络交换报文(message)相互通信。对于每对通信进程，通常将其中一个标为客户机(client)，另一个标识为服务器(server)。P2P结构中一个进程提供服务时为服务器，请求服务时为客户机。
应用层协议定义了运行在不同端系统上的应用程序进程如何相互传递报文，特别是：

  • 交换的报文类型。如请求报文和响应报文。
  • 各种报文类型的语法。如报文中各个字段及这些字段是如何描述的。
  • 字段的语义。即这些字段中包含的信息的含义。
  • 一个进程何时以及如何发送报文，以及对报文进行响应的规则。

一些常见的应用层协议如下：

应用层协议 网络应用 使用的传输层协议
SMTP 电子邮件 TCP
FTP 远程文件传输 TCP
Telnet 虚终端 TCP
HTTP Web服务 TCP
DNS 域名解析 TCP或UDP
SNMP 网络管理 UDP
RTP、SIP 流媒体传输 TCP或UDP

1. HTTP

20世纪90年代中期，万维网(World Wide Web)登上了舞台。Web的吸引力来自于按需操作。用户根据需要获取内容，相比之下，无线电广播和电视只能迫使用户收听。Web的应用层协议是超文本传输协议(HyperText Transfer Protocol, HTTP)，在[RFC 1945]和[RFC 2616]中进行了定义。HTTP由两个程序实现：一个客户端程序和一个服务器程序，它们运行在不同的端系统中，通过交换HTTP报文进行会话。HTTP定义了这些报文的结构以及客户和服务器进行报文交换的方式。
HTTP使用TCP作为运输层支撑协议。HTTP客户首先发起一个与服务器的TCP连接，一旦连接建立，浏览器和服务器可以通过套接字接口访问TCP。客户向套接字接口发送HTTP请求和接受HTTP响应报文，服务器向套接字接口接收HTTP请求报文和发送HTTP响应报文。一旦客户向套接字接口发送HTTP请求报文，该报文就脱离了客户控制并进入TCP控制。TCP为HTTP提供可靠数据传输服务，这意味着一个客户进程发出的每个HTTP请求报文最终能完整地到达服务器，即HTTP协议不用担心数据丢失，也不关注TCP从网络的数据丢失和乱序故障中恢复的细节。
HTTP是一个 无状态协议 ，HTTP服务器不保存关于客户的状态信息。如果某个特定客户在短时间内多次请求同一个对象，服务器会一直重新发送该对象，就像完全忘记不久之前所做的事情一样。

1.1 非持续连接和持续连接

HTTP即可以使用非持续连接(non-persistent connection)，即所有的请求及相应经相同的TCP连接发送；也可以使用持续连接(persistent connection)，即每个请求/响应对经一个单独的TCP连接发送。默认使用持续连接。
采用非持续连接的HTTP，会在每次访问一个资源的时候重新建立TCP连接。TCP连接需要三次握手，而且在客户和服务器中都要分配TCP的缓冲区和保持TCP变量，给服务器带来严重负担。但是使用并行连接可以缩短响应时间。
HTTP默认采用带流水线的持续连接。采用持续连接的情况下，服务器在发送响应后保持该TCP连接打开，在相同的客户与服务器之间的后续请求和响应报文能够通过相同的连接进行传送。一般来说，如果一条连接经过一定时间间隔仍未被使用，HTTP服务器就关闭该连接。

C. cookie

HTTP是无状态的，为了能够识别用户，其使用cookie([RFC 6265])技术。cookie技术有4个组件：

  • 在HTTP响应报文中有一个cookie首部行；
  • 在HTTP请求报文中有一个cookie首部行；
  • 在用户端系统中保留有一个cookie文件，并由用户浏览器进行管理；
  • web站点有一个后端数据库。

cookie的工作流程：

  1. 客户端访问一个网站，请求报文到达服务器
  2. 服务器喊声一个唯一识别码，以此识别码作为索引在数据库中产生一个表项
  3. 服务器用一个包含Set-cookie首部行的HTTP响应报文对客户端进行响应，Set-cookie首部行含有识别码
  4. 客户端接收到响应报文，看到Set-cookie首部，在它管理的特定cookie文件中添加报文中的识别码
  5. 客户端继续向服务器请求页面，此时浏览器会从文件中提取识别码并放置到请求报文的cookie首部行中
  6. 服务器根据请求报文中的cookie信息识别用户
  7. 客户端后续每次向服务器发送请求都将携带cookie信息直到cookie被删除

2. SMTP

简单邮件传输协议(SMTP, Simple Mail Transfer Protocol, RFC 5321)是因特网电子邮件中的主要的应用层协议。它使用TCP可靠数据传输服务，从发送方的邮件服务器向接收方的邮件服务器发送邮件。SMTP一般不使用中间邮件服务器发送邮件，当A向B发送邮件时，发起的TCP连接不经过其他服务器，如果B的服务器没有开机，那么邮件会保留在A的服务器上并在稍后继续进行新的发送尝试。
SMTP是如何将一个报文从发送邮件服务器传送到接收邮件服务器的呢？首先，客户SMTP在25号端口建立一个到服务器SMTP的TCP连接，如果服务器没有开机，客户会在稍后继续尝试连接。一旦连接简历，服务器和客户执行某些应用层的握手。在SMTP握手阶段，SMTP客户只是发送方的邮件地址和接收方的邮件地址。一旦该SMTP客户和服务器彼此介绍后客户发送该报文。SMTP依赖TCP提供的可靠数据传输无差错地将邮件投递到接收服务器。该客户如果有另外的报文要发送到该服务器，就在相同的TCP连接上重复这种处理；否则它只是TCP关闭连接。

3. 邮件访问协议

一旦SMTP将邮件报文从A的邮件服务器交付给B的邮件服务器，该报文就放入了B的邮箱中。目前主流的实现是用户通常在本地PC上运行一个用户代理程序，而邮件存储在总是保持开机的共享邮件服务器上，该邮件服务器与其他用户共享，并且通常由用户的ISP进行维护。
用户是如何通过运行于其本地PC上的用户代理获取ISP的邮件服务器上的邮件呢？首先肯定不能通过SMTP协议取回报文，因为去报文是一个拉(pull)操作，而SMTP是一个推(push)协议。目前有一些流行的邮件访问协议，包括POP3(Post Office Protocol——Version 3，第三版有据协议)、IMAP(Internet Mail Access Protocol，因特网邮件访问协议)以及HTTP。

A. POP3

POP3是一个极为简单的邮件访问协议，由[RFC 1939]定义。该协议非常简单，功能也有限。当用户代理打开一个到邮件服务器110端口上的TCP连接后，POP3就开始工作了。一共有三个阶段：特许、事务处理、更新。特许阶段，用户代理以明文形式发送用户名和口令。事务处理阶段用户代理取回报文，同时还能对报文做删除标记、取消报文删除标记、获取邮件统计信息。更新阶段是客户发出quit命令以结束该POP3会话，这是该邮件服务器会删除哪些被标记为删除的报文。
在特许阶段以后，用户代理仅使用4个命令：list、retr、dele、quit。支持的功能有限。

B. IMAP

POP3协议没有给用户提供任何创建远程文件夹并为报文指派文件夹的方法。为了解决这个或其他问题，有RFC 3501定义的因特网邮件访问协议应运而生，它比POP3具有更多特色，也比POP3复杂的多。IMAP现在的版本是”IMAP第7班第3次修订版(IMAP4REV1)”。IMAP一些特色包括：

  • 支持链接和断开两种操作模式。使用POP3时，客户端只会在一段时间内连接到服务器，指导下载完所有新信息即断开连接。IMAP中只要用户需要，客户端就会一直连接服务器。对于有很多或很大邮件的用户来说，使用IMAP4模式可以获得更快的响应时间。
  • 支持多个客户同时连接到一个邮箱。POP3协议假定邮箱当前的连接是唯一的连接，而IMAP4协议提供一种机制让客户能够感知其他当前连接到这个邮箱的用户的操作。
  • 支持访问消息中的MIME部分和部分获取。
  • 支持在服务器保留消息状态信息。
  • 支持在服务器上访问多个邮箱。
  • 支持服务器端搜索。
  • 支持一个定义良好的扩展机制。

IMAP4使用端口143在TCP/IP连接上工作。

3. DNS

域名系统(DNS, Domain Name Service)是互联网的一项服务，它作为一个将域名和IP地址相互映射的一个分布式数据库，能够诗人更方便的访问互联网。DNS使用TCP和UDP的53号端口。当前对于每一级域名长度的限制是63字符，域名总长度不能超过253字符。
除了主机名到IP地址的转换外，DNS还提供了一些重要服务：主机别名、邮件服务器别名、负载分配。
