字符串排序算法

发表于 2019-09-25 | 更新于 2020-05-17 | 分类于 数据结构与算法

字符串排序算法主要运用了字符串的特殊性来优化并达到通用型排序算法达不到的时间复杂度。基于比较的排序的最有时间复杂度是O(nlgn)，而radix sort能达到O(n)。

首先给出一个工具类，以打印数组。

1. 键索引计数法 key-index counting

键索引计数法是一种适用于小整数键的简单排序方法，同时也是后续算法的基础。这个算法有四个步骤：

  1. 频率统计。计算出各字符的个数
  2. 将频率转换为索引。
  3. 数据分类。
  4. 回写。将排序好的数组替换原先数组

2. 低位优先的字符串排序 LSD radix sort

LSD radix sort是基于key-index counting的一种对等长字符串数组排序的高效算法。其原理就是利用key-index counting算法，对要排序的字符串进行逐字符排序，直到字符串中每个字符都排序完成。顺序是从字符串最后一个开始，一直到最后一个字符。因为key-index counting是稳定的，所以可以从后至前排序后最后结果是有序的。算法代码如下：

3. 高位优先的字符串排序 MSD radix sort

LSD适用于排序等长字符的数组，但是显示应用中，变长数组间的排序也很常见。MSD radix sort(most-significant-digit-first)，是从每个字符传的第一个字符开始排序，一直到最后一个字符。该算法的步骤是：

  1. 利用key-index counting对第一个字符进行排序
  2. 将首字符相同的子数组拿出来，并去掉第一个已经排好序的字符。然后重复第一步，直至排序完成

MSD算法的性能取决于数据。对于随机输入，MSD只会检查足以区别字符串所需的字符。相对于数据数据中的字符总数，算法的运行时间是亚线性的（它只会检查输入字符中的一小部分）。对于非随机输入，MSD可能是亚现行的，但需要检查的字符可能会更多；特别是对相等的键，需要检查所有字符。所以存在大量等值键时它需要的运行时间是接近线性的。在最坏情况下(所有键都相同)，MSD会检查所有字符，时间复杂度是线性的。
要将基于大小为R的字符表的N个字符串排序，MSD算法平均需要检查N * logN / logR个字符。
由于该算法循环调用sort函数，并且sort函数里面会创建count[]数组，所以该算法有以下问题：

  • 当小数组过多时，该算法会创建大量的count[]数组，此时会对运行时间和内存造成比较大的影响，使算法性能不高，所以在数组个数变小时，推荐使用插入排序
  • 当数组有过多的相同的字符串或者子字符串时，算法需要一直比对到最后一个字符或者比较较多的字符，此时算法的性能也会偏低
  • 该算法需要较多的内存，比如aux[]数组和count[]数组的创建，都是比较消耗内存的

4. 三向字符串快速排序 3-way string quicksort

3-way string quicksort 是利用了快排和MSD的想法并进行合并而出来的一个算法，主要是解决了MSD radix sort的三个性能可能受影响的问题，在不需要大量内存并且当字符串出现大量相同的情况下还能继续保持良好的时间复杂度。

和快排一样，最好在排序之前将数组打乱或是将第一个元素与一个随机位置的元素交换以得到一个随机的切分元素。这么做是为了预防数组有序或接近有序的最坏情况。
